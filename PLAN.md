# Frankfurt History Web App — Plan

## Goal

A static web app for exploring the Frankfurt History archive on an interactive map. Supports layer filtering, German/English, deep-linkable articles, walking navigation between locations, and browser-native TTS article reading.

## Tech Stack

| Concern | Choice | Why |
|---------|--------|-----|
| Framework | **TanStack Start** (React) | SSG-capable, type-safe routing with first-class URL search param state — ideal for map/layer/filter state in the URL |
| Map | **MapLibre GL JS** via `react-map-gl` | Free Mapbox GL fork, vector tiles, clustering; react-map-gl gives declarative React bindings |
| Tiles | **Self-hosted PMTiles** | Protomaps basemap extract for the Frankfurt area (~67 MB), custom warm-tone style via `protomaps-themes-base`. No tile server, no API key. |
| Routing | **OpenRouteService** | Free tier (40 req/min, 500/day), walking profile; nearest-neighbor routes pre-cached at build time via `precache_routes.py` |
| TTS | **Web Speech API** | Browser-native, zero-bundle-cost. Supports German and English with play/pause/speed controls. |
| Translation | **API-native** (Accept-Language header) | The API already serves English translations; archive both `de` and `en` at fetch time. Override system allows hand-correcting any translation. |
| Styling | **Tailwind CSS v4** | Utility-first, small bundle |
| Hosting | **Cloudflare Pages** + **R2** | Global edge, free tier generous (100k req/day); R2 for images (10 GB free, zero egress) |

## Data Pipeline (build time)

1. **`archive.py`** — fetches API data into `data/de/<theme>/<poi>.md` and `data/en/<theme>/<poi>.md` + images. Requests each language via `Accept-Language` header.
2. **`scripts/merge.py`** — for each locale, deep-merges `data/<lang>/` with `overrides/` into `content/<lang>/`.
3. **`scripts/geojson.py`** — reads merged content frontmatter, emits one `<theme>.geojson` per theme. Filters out POIs with coordinates outside the Frankfurt bounding box.
4. **`scripts/precache_routes.py`** — for each POI, finds the 3 nearest POIs within the same theme (haversine), optionally fetches walking routes from ORS (with `ORS_API_KEY`), writes to `app/public/data/routes/<theme>/<poi-slug>.json`.
5. **`pmtiles extract`** — extracts Frankfurt area (bbox 8.3,49.9–9.0,50.3, maxzoom 15) from Protomaps daily build into `app/public/frankfurt.pmtiles`.
6. TanStack Start pre-renders both `de/` and `en/` routes at build time.

## Override System

The `overrides/` directory is committed by hand and **never touched by the cron**. It lets you fix coordinates, rewrite texts, correct translations, or add missing metadata without losing changes when `archive.py` re-fetches from the API.

An override file mirrors the path of its `data/` counterpart (including the `de/` or `en/` prefix). It can be **partial** — you only include the frontmatter fields or body sections you want to change.

| Scenario | Result |
|----------|--------|
| No override exists | `data/<lang>/` file used as-is |
| Override has only frontmatter | Frontmatter fields merged, body from `data/` |
| Override has frontmatter + body | Frontmatter merged, body from override |
| Override sets a field to `null` | Field removed from merged output |
| `data/` file deleted by API | Override-only POI still appears (allows adding custom locations) |
| DE override exists, no EN override | Structural fields (coordinates, categories, filters) applied to both locales |

## Features

### Map View
- Full-screen MapLibre map centered on Frankfurt (50.11, 8.68), zoom ~13
- Self-hosted PMTiles basemap with warm archival color palette
- Map constrained to Frankfurt bounding box (8.4–8.9 lng, 50.0–50.25 lat)
- POIs rendered as clustered circle markers, colored by theme
- User location via GeolocateControl (blue dot + "center on me")
- Map position tracked in URL search params (`lat`, `lng`, `z`)

### Layer Picker
- Themes as toggleable layers with colored dots and POI counts
- "Show all / hide all" toggle
- Layer state persisted in URL: `?layers=1,3`

### Article Panel
- Desktop: 420px right-side panel with slide-in animation
- Mobile: bottom sheet with peek/half/full snap points, drag to resize
- Deep link URL: `/$lang/$theme/$slug` — flies to POI immediately from frontmatter coordinates
- Image galleries with lightbox, before/after sliders
- Share (Web Share API + clipboard fallback) and print support

### Navigation
- **"Navigate here"** — browser geolocation → ORS live walking route → dashed sepia line on map with turn-by-turn steps. Falls back to Google Maps link.
- **Nearby POIs** — pre-cached 3 nearest neighbors per POI with distance/time, one-tap navigation between them.

### Language Toggle
- `de` / `en` toggle in header
- Preserves all URL state (map position, layers, active article)

### TTS
- Web Speech API with play/pause/stop, adjustable speed (0.75×–1.5×)
- Language-aware voice selection (German or English)

## URL Scheme

| URL | View |
|-----|------|
| `/` | Redirects to `/de/` |
| `/de/` | Map with all layers, German UI |
| `/en/` | Map with all layers, English UI |
| `/de/frankfurt-und-der-ns/0017-hochbunker` | Article + map centered on POI |
| `/de/?layers=1,3` | Map with specific layers active |
| `/de/?lat=50.09&lng=8.67&z=15` | Map at specific position/zoom |

## Visual Design

### Direction: "Archival Cartography"

A visual language that feels like studying a well-curated museum map — not a tech product. The map is the hero; UI recedes until needed.

### Color System

| Role | Value | Usage |
|------|-------|-------|
| Paper | `#FAF8F5` | Background, panels, cards |
| Ink | `#1A1A1A` | Primary text |
| Faded Ink | `#6B6560` | Secondary text, metadata, captions |
| Sepia | `#8B7355` | Accent, active states, links |
| Sepia Light | `#D4C5AD` | Borders, dividers, inactive markers |
| Red Oxide | `#A0522D` | Alert accent, active POI marker |

Each theme gets its own marker color from muted earth tones.

### Typography

- **Headings**: Libre Baskerville (serif)
- **Body/UI**: Inter (sans-serif)

## R2 Asset Strategy

| Asset | Size | Upload strategy |
|-------|------|-----------------|
| POI images (originals) | ~3.5 GB | Synced by `archive.py` via `wrangler r2 object put` |
| PMTiles (Frankfurt area) | ~67 MB | Re-generated occasionally via `pmtiles extract` |

Images are **not stored in git** — they live only on R2. `merge.py` rewrites image paths to R2 URLs at build time, using Cloudflare's `cdn-cgi/image/` edge transform for responsive sizing.
